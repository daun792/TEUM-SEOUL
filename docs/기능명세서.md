# 틈서울 기능 명세서

## 1. 서비스 개요

공연·축제 전후 30~180분의 남는 시간을 입력하면 공연장 좌표와 서울 공공데이터 장소 좌표를 비교하여 실제 장소 1~2개의 짧은 코스를 생성하고, 익명 후기를 공유하는 서비스다.

## 2. 화면

| 화면 | 주요 기능 |
|---|---|
| 홈 | 서비스 소개, 추천 시작, 제공 공연 일부, 최근 후기 |
| 조건 입력 | 공연 검색, 공연 전·후, 30~180분, 선호 카테고리 |
| 추천 결과 | 장소 1~2개, 이동·체류시간, 복귀 버퍼, 공유, 후기 작성 |
| 게시판 목록 | 검색, 카테고리 필터, 조회수·좋아요 표시 |
| 게시글 상세 | 조회, 좋아요, 북마크, 수정·삭제 |
| 글 작성·수정 | 익명 후기와 수정용 비밀번호 입력 |
| 내 북마크 | 현재 브라우저에 저장한 게시글 목록 |
| 챗봇 | 실제 장소·게시글 기반 질의응답, 플로팅·모바일 전체 화면 |

## 3. 추천 정책

### 입력

- 공연 ID
- 시점: `before` 또는 `after`
- 남는 시간: 30~180분
- 카테고리: 문화시설, 관광지, 쇼핑, 레포츠, 상관없음

### 이동시간

```text
예상 보행거리 = Haversine 직선거리 × 1.25
예상 보행시간 = 예상 보행거리 ÷ 4.5km/h
```

### 체류시간

짧은 시간에도 결과를 제공하기 위해 입력 시간별 체류시간 상한을 적용한다.

| 입력 시간 | 장소별 체류시간 상한 |
|---|---:|
| 30~45분 | 10분 |
| 46~60분 | 20분 |
| 61~90분 | 35분 |
| 91~180분 | 쇼핑 30분, 관광지 40분, 문화시설 50분, 레포츠 60분 |

### 장소 수

- 30~90분: 1곳
- 91~180분: 2곳 우선, 시간 초과 시 1곳
- 공연 전: 출발·방문·공연장 복귀 시간에 15분 버퍼를 더해 입력 시간 이하인지 검증
- 공연 후: 공연장 출발 후 방문 시간 합계가 입력 시간 이하인지 검증
- 선호 카테고리에서 결과가 없으면 전체 추천 카테고리로 완화하고 화면에 알림

## 4. 커뮤니티

- 로그인 없음
- 카테고리: 공연 전 코스, 공연 후 코스, 장소 후기, 자유
- CRUD 및 검색
- 수정·삭제 시 작성 비밀번호 비교
- 요구사항에 따라 비밀번호는 평문 저장·비교하며 실무 서비스에는 부적절함을 UI에 안내
- 상세 조회 시 조회수 증가

## 5. 좋아요·북마크

### 좋아요

- 브라우저 최초 접속 시 `crypto.randomUUID()`로 `client_id` 생성
- `localStorage`에 저장
- `(post_id, client_id)` DB 고유 제약
- 다시 누르면 취소되는 toggle

### 북마크

- `localStorage.teumSeoulBookmarks`에 게시글 ID 배열 저장
- 서버 회원정보와 연결하지 않음
- 다른 브라우저·기기와 동기화하지 않음
- 삭제된 게시글은 북마크 화면 조회 시 자동 제거

## 6. 공유

공유 URL 예시:

```text
/course/shared?performanceId=6318&phase=after&minutes=120&category=문화시설&placeIds=120,501
```

- Web Share API 지원 시 기기 공유창
- 미지원 시 클립보드 복사
- 백엔드가 공연·장소 ID와 시간 범위를 재검증
- 유효하지 않은 장소: 404
- 시간 또는 파라미터 오류: 422

## 7. 챗봇

- 장소명, 주소, 카테고리와 게시글 제목·내용을 검색
- 검색된 실제 레코드만 OpenAI 프롬프트의 근거로 전달
- 목록에 없는 장소, 운영시간, 행사 일정을 만들지 않도록 시스템 지시
- API 키 미설정·오류 시 검색 결과를 템플릿 문장으로 반환

## 8. 제외 기능

지도, 날씨, 실시간 길찾기, 실제 공연 일정 검증, 운영시간 검증, 예약·결제, 회원가입, 댓글, 실시간 알림, 계정 간 북마크 동기화는 구현하지 않는다.
