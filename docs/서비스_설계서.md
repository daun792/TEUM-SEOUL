# 틈서울 서비스 설계서

## 1. 프로젝트명
**틈서울 — 공연 전후 남는 시간을 채워주는 초단기 서울 코스 커뮤니티**

## 2. 한 문장 설명
축제·공연 전후 남는 시간과 사용자의 선호 카테고리를 입력하면, 제공된 서울 공공데이터를 이용해 1~2개의 장소로 구성된 짧은 코스를 추천하고 익명으로 후기를 공유하는 서비스다.

## 3. 문제 정의
공연 관람객은 공연 시작 전이나 종료 후 30분~3시간이 남아도 공연장 주변에서 방문할 장소를 찾고, 이동시간과 체류시간을 계산해 짧은 일정을 만드는 데 시간이 많이 든다. 틈서울은 공연을 위치 기준점으로 삼아 남는 시간 안에 들어오는 장소만 추천한다.

## 4. 데이터 범위
실제 ZIP에 포함된 서울 JSON은 7종, 총 6,518건이다.

| 유형 | 건수 | 활용 |
|---|---:|---|
| 축제공연행사 | 201 | 공연 검색 및 코스 출발점 |
| 문화시설 | 566 | 추천 후보 |
| 쇼핑 | 4,368 | 추천 후보 |
| 관광지 | 783 | 추천 후보 |
| 레포츠 | 126 | 추천 후보 |
| 숙박 | 423 | 챗봇 조회 가능, 추천 제외 |
| 여행코스 | 51 | 챗봇 조회 가능, 추천 제외 |

`SOURCE.md`에는 음식점 1,632건을 포함한 총 8,150건으로 기재되어 있으나 실제 ZIP에는 `서울_음식점.json`이 없다. 서비스는 실제 파일만 사용하며 음식점에 의존하지 않는다.

## 5. 사용자 입력
- 공연·축제 검색 및 선택
- 시점: 공연 전 / 공연 후
- 남는 시간: 30~180분
- 선호 카테고리: 문화시설 / 관광지 / 쇼핑 / 레포츠 / 상관없음

## 6. 추천 결과
- 실제 공공데이터 장소 1~2개
- 공연장과 장소 간 직선거리
- 예상 도보시간
- 장소별 권장 체류시간
- 공연 전·후 방문 타임라인
- 추천 사유
- 데이터·거리 추정 안내문
- 재현 가능한 공유 URL
- 링크 복사 및 기기 기본 공유 버튼

### 6.1 추천 결과 공유
공유 URL은 다음 값을 쿼리 문자열에 포함한다.

```text
/course/shared
?performanceId=123
&phase=after
&minutes=120
&category=문화시설
&placeIds=501,830
```

공유 링크를 열면 프론트엔드는 `GET /api/courses/shared`를 호출한다. 백엔드는 공연 ID, 장소 ID, 시간 범위를 검증하고 현재 데이터로 타임라인을 재구성한다. `navigator.share()`를 지원하는 기기에서는 기본 공유 창을 열고, 미지원 환경에서는 `navigator.clipboard.writeText()`로 URL을 복사한다.

## 7. 추천 알고리즘

### 7.1 거리 계산
공연과 후보 장소의 위경도로 Haversine 직선거리를 계산한다.

예상 도보시간은 다음 규칙을 사용한다.

```text
예상 도보거리 = 직선거리 × 1.25
예상 도보시간 = 예상 도보거리 ÷ 4.5km/h
```

실제 도로 경로가 아니므로 결과에 ‘직선거리 기반 예상값’이라고 표시한다.

### 7.2 권장 체류시간
- 쇼핑: 30분
- 관광지: 40분
- 문화시설: 50분
- 레포츠: 60분

### 7.3 코스 구성
- 30~90분: 기본 1곳
- 91~180분: 조건이 맞으면 2곳
- 공연 전: `공연장 → 장소 1 → 장소 2 → 공연장`의 총시간과 15분 복귀 버퍼가 입력 시간 이하여야 한다.
- 공연 후: `공연장 → 장소 1 → 장소 2`의 총시간이 입력 시간 이하여야 한다.
- 선택 카테고리 후보가 없으면 거리순으로 다른 카테고리를 탐색하고 완화 사실을 표시한다.

### 7.4 추천 점수
- 시간 적합성: 40점
- 가까운 거리: 35점
- 선호 카테고리 일치: 20점
- 이미지·주소 정보 보유: 5점

## 8. 화면 구성

### 홈
- 서비스 소개
- ‘틈 채우기’ 시작 버튼
- 최근 후기
- 챗봇 플로팅 버튼

### 공연·조건 입력
- 공연명 검색 자동완성
- 공연 전·후 선택
- 남는 시간 입력
- 선호 카테고리 선택

### 추천 결과
- 공연 정보
- 남는 시간 요약
- 장소 1~2개 카드
- 공연 전·후 타임라인
- 총 예상 이동·체류시간
- 결과 복사 및 후기 작성 연결

### 익명 커뮤니티
- 카테고리: 공연 전 코스 / 공연 후 코스 / 장소 후기 / 자유
- 목록·검색·상세·작성·수정·삭제
- 조회수
- 익명 좋아요
- 브라우저 북마크 및 ‘내 북마크’ 목록
- 수정·삭제 비밀번호 확인

### 좋아요
최초 접속 시 프론트엔드가 `crypto.randomUUID()`로 익명 `client_id`를 생성해 `localStorage`에 저장한다. 좋아요 요청은 `(post_id, client_id)` 고유 제약을 가진 `post_likes` 테이블에 반영한다. 동일 브라우저에서 다시 누르면 좋아요가 취소되는 toggle 방식으로 동작한다.

### 북마크
북마크는 서버 계정 없이 `localStorage`의 `teumSeoulBookmarks` 배열에 게시글 ID를 저장한다. ‘내 북마크’ 화면은 저장된 ID를 `GET /api/posts?ids=1,2,3`으로 조회한다. 브라우저 데이터 삭제 시 북마크도 삭제되며 다른 기기와 동기화되지 않는다.

### 챗봇
- 공연 주변 장소 검색
- 남는 시간에 맞는 장소 질의
- 커뮤니티 후기 검색
- 실제 DB 검색 결과만 OpenAI에 전달
- OpenAI 장애 시 템플릿 응답

## 9. 데이터 모델

### locations
`id`, `content_id`, `name`, `address`, `latitude`, `longitude`, `category`, `image_url`, `district`, `source`

축제공연행사도 동일한 테이블에 `category='축제공연행사'`로 저장한다.

### posts
`id`, `title`, `content`, `category`, `password`, `performance_name`, `available_minutes`, `view_count`, `like_count`, `created_at`, `updated_at`

### post_likes
`id`, `post_id`, `client_id`, `created_at`

`(post_id, client_id)`에 고유 제약을 적용한다. 북마크는 사용자별 개인 상태이므로 SQLite에 저장하지 않고 브라우저 localStorage에서 관리한다.

## 10. API
- `GET /api/health`
- `GET /api/performances?q=`
- `GET /api/places?category=&q=`
- `POST /api/courses/recommend`
- `GET /api/courses/shared?performance_id=&phase=&minutes=&category=&place_ids=`
- `GET /api/posts?q=&category=&ids=`
- `GET /api/posts/{id}`
- `POST /api/posts`
- `PUT /api/posts/{id}`
- `DELETE /api/posts/{id}`
- `POST /api/posts/{id}/likes/toggle`
- `POST /api/chat`

## 11. 오류 처리
- 공연 미선택 또는 시간 범위 오류: HTTP 422
- 공연 데이터 없음: HTTP 404
- 시간 내 후보 없음: 빈 결과가 아니라 조건 완화 결과 또는 명확한 안내 반환
- 게시글 비밀번호 불일치: HTTP 403
- OpenAI 오류: 검색된 장소·게시글을 템플릿 문장으로 반환
- 데이터 적재 실패: 서버 시작 중 오류 로그 출력 후 비정상 종료

## 12. 선택 기능
외부 API가 필요 없는 다음 기능을 구현한다.

1. 게시판 제목·내용 검색, 카테고리 필터, 조회수
2. 추천 결과 공유 URL, 링크 복사, Web Share API
3. 게시글 익명 좋아요와 브라우저 북마크

## 13. 제외 범위
지도, 날씨, 실시간 길찾기, 실제 공연 일정 검증, 운영시간 확인, 예약·결제, 회원가입, 댓글, 실시간 알림, 계정 간 북마크 동기화는 구현하지 않는다.

## 14. 완료 기준
- 대표 입력 10개에서 실제 JSON 장소 1~2개 추천
- 추천 총시간이 사용자의 남는 시간 이하
- 공연 전 코스에 15분 복귀 버퍼 반영
- 게시판 CRUD·검색·조회수 정상 동작
- 동일 브라우저에서 게시글 좋아요가 중복 누적되지 않음
- 북마크가 새로고침 후 유지되고 북마크 목록에서 재조회됨
- 공유 URL을 새 탭에서 열었을 때 동일 공연·장소·타임라인이 표시됨
- Web Share API 미지원 환경에서 링크 복사가 정상 동작함
- 챗봇이 DB에 없는 장소를 생성하지 않음
- Netlify·Render 외부 URL 정상 동작
- `.env`가 Git 저장소에 포함되지 않음
- 기능 명세서에 출처·라이선스·실제 파일 건수·데이터 한계를 기재
